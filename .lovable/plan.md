# Plan: Supabase Credentials aus Code entfernen

## Ziel
Hardcodierte Supabase URL und Publishable Key in `src/integrations/supabase/client.ts` durch `import.meta.env`-Zugriffe ersetzen, mit klaren Fehlermeldungen bei fehlenden Variablen. Zusätzlich `.gitignore` und `.env.example` pflegen.

## Hinweis zur Umgebung
- `.env` existiert bereits im Projekt-Root und enthält die korrekten Werte (`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_SUPABASE_PROJECT_ID`) — Lovable Cloud injiziert diese bei jedem Build automatisch. Die App läuft also nach der Umstellung weiter.
- `.gitignore` enthält aktuell **keinen** `.env`-Eintrag → wird ergänzt.
- `.env.example` existiert noch nicht → wird neu erstellt.

## Änderungen

### 1. `src/integrations/supabase/client.ts`
String-Literale durch env-Lookup mit Validierung ersetzen:

```ts
import { createClient } from '@supabase/supabase-js';
import type { Database } from './types';

const SUPABASE_URL = import.meta.env.VITE_SUPABASE_URL as string | undefined;
const SUPABASE_PUBLISHABLE_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY as string | undefined;

if (!SUPABASE_URL) {
  throw new Error('VITE_SUPABASE_URL is not set');
}
if (!SUPABASE_PUBLISHABLE_KEY) {
  throw new Error('VITE_SUPABASE_PUBLISHABLE_KEY is not set');
}

export const supabase = createClient<Database>(SUPABASE_URL, SUPABASE_PUBLISHABLE_KEY, {
  auth: {
    storage: localStorage,
    persistSession: true,
    autoRefreshToken: true,
  },
});
```

Kommentar „This file is automatically generated" wird entfernt, da die Datei bewusst manuell editiert wird.

Ergänzen am Ende:

## Nicht im Scope
- Keine Änderung an Edge Functions (nutzen `Deno.env`).
- Keine Änderung an anderen Stellen, die `import.meta.env.VITE_SUPABASE_*` bereits korrekt lesen (z. B. `src/hooks/useChat.ts`).
- Der hardcodierte `supabase_anon_key` in der DB-Funktion `trigger_translate_new_activity` bleibt unangetastet (separater Kontext, Server-seitig).

## Hinweis zur Sicherheit
Der Publishable/Anon Key ist kein Secret — er ist für den Client-Einsatz gedacht und durch RLS abgesichert. Die Umstellung verbessert dennoch die Konfigurierbarkeit (verschiedene Environments) und entfernt Duplizierung zwischen `.env` und Code.
